# Источник цены в AI-сметчике

## Цель

Пользователь всегда видит, какая цена фактически использована при генерации позиции AI-сметы. Система не скрывает замену отсутствующей региональной цены базовой ценой ФСБЦ.

## Контракт данных

Для выбранной нормы backend возвращает сводку ценовых источников и список ресурсов:

- источник: `regional` или `fsbc_base`;
- идентификатор и версия каталога;
- код и наименование ресурса;
- единица измерения;
- применённая цена;
- признак замены отсутствующей региональной цены.

Данные формируются из закреплённых `resource_price_id` и доказательной части снимка цены. Frontend не определяет источник по косвенным признакам и не пересчитывает сумму.

## Интерфейс

В карточке выбранной нормы показывается строка «Использованные цены»:

- только региональные цены — нейтральная метка «Региональные цены»;
- есть замены — предупреждение «Для N ресурсов применены базовые цены ФСБЦ»;
- раскрываемый список показывает код, название, единицу, цену и источник каждого ресурса.

В блоке готовности выводится общий необязательный warning с количеством позиций, где применена ФСБЦ. Полное отсутствие цены остаётся обязательным блокером.

## Совместимость и ошибки

Новые поля опциональны для ранее созданных сессий. При их отсутствии UI не падает и не угадывает источник. Позиция с валидной базовой ценой ФСБЦ может быть применена; позиция без доказуемой цены — нет.

## Проверка

- backend-тест источника цены и сериализации;
- тест нормализатора API на отсутствие и наличие новых полей;
- Vitest карточки нормы для регионального, смешанного и отсутствующего источника;
- production smoke-test сессии №58 после получения рассчитанных норм.
