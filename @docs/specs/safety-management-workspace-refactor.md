# Рефакторинг рабочего пространства охраны труда

## Цель

Модуль охраны труда должен открываться с карточек сотрудников и вести пользователя по понятному рабочему циклу: сотрудники, инструктажи, наряды, проверки, нарушения, документы и правила допуска.

## Проблема

До рефакторинга админка строила карточки сотрудников только из записей охраны труда. Если сотрудник есть в HR-модуле, но по нему еще не создан инструктаж, медосмотр, СИЗ или правило допуска, экран показывал пустое состояние и пользователь не понимал, с чего начинать.

## Контракт API

Добавлен read-only endpoint:

`GET /api/v1/admin/safety-management/employee-cards`

Параметры:

- `project_id` - текущий проект, применяется к проектным требованиям допуска.
- `employee_id` - точечная карточка сотрудника.
- `search` - поиск по ФИО или табельному номеру.
- `per_page` - лимит карточек, максимум 200.

Ответ:

- `cards[]` - карточки всех найденных сотрудников организации.
- `summary.total` - всего сотрудников.
- `summary.ready` - готовы к работам.
- `summary.attention` - карточку нужно заполнить.
- `summary.blocked` - есть блокеры допуска.

Карточка содержит ФИО, табельный номер, статус занятости, текущую должность/подразделение/объект, статус готовности, следующее действие, блокеры, предупреждения и последние записи по требованиям, инструктажам, медосмотрам и СИЗ.

## UX-решение

Стартовая рабочая зона: `Сотрудники`.

Навигация больше не использует формулировки `Сегодня`, `Старт`, `Шаг`. Разделы называются по рабочим зонам:

- Сотрудники
- Инструктажи
- Наряды
- Проверки
- Нарушения
- Документы
- Правила допуска

На первом экране пользователь видит:

- KPI по сотрудникам, неподготовленным карточкам и подписям.
- Обзор рабочего цикла.
- Приоритетные действия.
- Карточки сотрудников с поиском и фильтрами готовности.

## Проверки

- Карточки сотрудников должны отображаться даже при пустых журналах охраны труда.
- В UI не должно быть технических стартовых формулировок вроде `Сегодня`, `Старт`, `Шаг`.
- Старые журналы и таблицы остаются доступны как детализация, но не являются первым пользовательским ориентиром.
