# Модуль «Охрана труда»: статусная модель

## Статусы допуска

- `admitted`: сотрудник допущен, критических блокеров нет.
- `partial`: сотрудник может быть допущен с ограничениями или предупреждениями, например документ скоро истечет.
- `not_admitted`: сотрудник не допущен, есть критический блокер.
- `pending`: требуется проверка или данных недостаточно для итогового решения.

## Блокирующие причины

- Сотрудник неактивен или уволен.
- Не настроена матрица требований для работ повышенного риска.
- Нет обязательного обучения или срок действия истек.
- Нет обязательного инструктажа или целевого инструктажа.
- Нет медосмотра, срок истек, результат отрицательный.
- Не выданы обязательные СИЗ или срок действия истек.
- Нет активного наряда-допуска для работ, где он обязателен.

## Предупреждения

- Обучение, медосмотр или СИЗ скоро истекают.
- Медосмотр разрешает работу только с ограничениями.
- Есть открытые нарушения или корректирующие действия по объекту.
- Есть неполные данные, которые не блокируют текущий сценарий, но требуют внимания специалиста ОТ.

## Статусы записей

### Требование сотрудника

- `valid`: требование закрыто действующей записью.
- `expired`: срок действия истек.
- `revoked`: запись отозвана.
- `waived`: исключение согласовано по регламенту.

### Обучение

- `passed`: обучение или проверка знаний пройдены.
- `failed`: проверка не пройдена.
- `pending`: результат ожидается.

### Медосмотр

- `fit`: годен.
- `fit_with_restrictions`: годен с ограничениями.
- `not_fit`: не годен.

### СИЗ

- `issued`: СИЗ выдано и действует.
- `returned`: СИЗ возвращено.
- `lost`: СИЗ утрачено.
- `damaged`: СИЗ повреждено.

## Итоговое правило

Если есть хотя бы один критический блокер, итоговый статус `not_admitted`. Если критических блокеров нет, но есть предупреждения или ограничения, итоговый статус `partial`. Если требований нет или все требования закрыты действующими записями, статус `admitted`.
