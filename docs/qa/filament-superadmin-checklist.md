# QA-чеклист Filament-суперадминки

Использовать на staging или локальном окружении с доступным `/admin`. Не запускать dev-сервер только ради этой проверки.

## Доступ и роли

- [ ] Войти как `super_admin`.
- [ ] Убедиться, что доступен дашборд и все основные группы навигации.
- [ ] Войти как `security_auditor`.
- [ ] Убедиться, что критичные действия создания, изменения и удаления недоступны.
- [ ] Проверить, что биллинговые и пользовательские виджеты не видны ролям без нужных permissions.

## Дашборд

- [ ] Проверить виджеты выручки, пользователей, тарифов, здоровья платформы, роста, рисков и доставки уведомлений.
- [ ] Убедиться, что виджеты не показывают секреты и служебные конфигурационные значения.
- [ ] Проверить пустые или нулевые состояния без поломки верстки.

## Организации

- [ ] Открыть список организаций.
- [ ] Проверить фильтры по статусу, тарифу, просрочке, пробному периоду и модулям.
- [ ] Открыть карточку организации.
- [ ] Проверить профиль, подписку, пользователей, модули, платежи и активность.
- [ ] Выполнить приостановку организации с причиной.
- [ ] Выполнить реактивацию организации.
- [ ] Проверить события в аудите.

## Пользователи

- [ ] Открыть профиль пользователя.
- [ ] Проверить организации, роли, статус входа и подтверждение email.
- [ ] Убедиться, что прямое изменение пароля не доступно.
- [ ] Выполнить безопасное действие блокировки или разблокировки при наличии прав.

## Биллинг

- [ ] Открыть тарифы, подписки, пакеты организаций и платежи.
- [ ] Убедиться, что платежи read-only.
- [ ] Проверить masked provider identifiers.
- [ ] Выполнить разрешенное действие подписки только через кнопку workflow.
- [ ] Проверить событие аудита.

## Блог

- [ ] Создать черновик статьи.
- [ ] Проверить автосохранение черновика.
- [ ] Заполнить заголовок, адрес, лид, тело, автора, категорию, обложку и SEO.
- [ ] Проверить редакционный чеклист.
- [ ] Проверить SEO-превью.
- [ ] Загрузить поддерживаемый медиафайл.
- [ ] Проверить, что неподдерживаемый файл отклоняется.
- [ ] Опубликовать статью при заполненном чеклисте.
- [ ] Вернуть статью в черновик.
- [ ] Проверить историю версий и восстановление черновика.

## Уведомления

- [ ] Открыть шаблон уведомления.
- [ ] Проверить предпросмотр на тестовых данных.
- [ ] Отправить тест только текущему системному администратору.
- [ ] Открыть диагностику доставки.

## Поддержка и мониторинг

- [ ] Открыть обращения поддержки.
- [ ] Назначить оператора.
- [ ] Изменить статус обращения.
- [ ] Добавить внутреннюю заметку.
- [ ] Эскалировать обращение.
- [ ] Открыть ошибки приложения.
- [ ] Изменить статус ошибки приложения.
- [ ] Проверить, что полный стек и чувствительный контекст не выводятся в UI.

## Аудит

- [ ] Открыть журнал аудита.
- [ ] Отфильтровать по организации, субъекту, действию и периоду.
- [ ] Проверить маскирование чувствительных деталей.
- [ ] Убедиться, что ресурс read-only.
