# AI-сметчик: handoff после реализации no-air этапа

Дата: 2026-06-29.

## Что уже реализовано

- Генерация больше не должна заполнять смету абстрактными строками, если нет нормального основания: generic-комплексы, нулевые цены и неподтвержденные нормы блокируются проверкой.
- Входные файлы классифицируются как документация, планировка/чертеж, спецификация, скан или CAD-файл. Для страниц сохраняется профиль понимания документа.
- Из OCR-текста планировок извлекаются площади помещений, двери, инженерные трассы и агрегированные объемы для полов, потолков, стен и проемов.
- Из спецификаций/ведомостей извлекаются количественные позиции: светильники, радиаторы, трубы, двери и похожие строки.
- Для каждого извлеченного объема добавлен evidence graph: откуда взят объем, сколько ссылок на источник, уверенность и требуется ли ручная проверка.
- DWG/DXF принимаются и безопасно классифицируются, но система не делает вид, что извлекла геометрию, если CAD-разборщик не подключен.
- Проверка сметы блокирует позиции с нулевой ценой, отсутствующим источником цены, небезопасной нормой или отсутствием ресурсов нормы.
- В админке добавлен рабочий сценарий проверки: фильтр проблемных позиций, вход в мастер по позиции, выбор подходящей нормы/цены и сохранение выбора.
- Выбор пользователя записывается в learning loop: выбранная норма становится положительным сигналом, замененная норма фиксируется как отклоненная.
- Обработка обучающих наборов ограничена очередью, rate limit и уникальными lock-ключами, чтобы несколько тяжелых задач не давили backend одновременно.

## Как работает загрузка обучающего набора

Форма обучающего набора принимает эталонную смету, проектную документацию, чертежи, сканы и ведомости. Если включить обработку после сохранения, создается задача обработки набора: система сохраняет файлы как эталонный контекст, классифицирует документы и готовит данные для сопоставления будущих генераций.

Это не мгновенное переобучение большой модели в момент загрузки файла. Корректная схема такая:

1. Загружается эталон: смета из Гранд-Сметы или другой проверенный расчет плюс исходные документы.
2. Система извлекает структуру, объемы, нормы, источники и связи "документ -> объем -> норма -> цена".
3. Эти связи используются как обучающие примеры, сигналы ранжирования и контроль качества для следующих генераций.
4. Ручные выборы сметчика в мастере добавляют короткие обучающие сигналы: что выбрали, что отклонили и на каком основании.
5. Отдельное полноценное дообучение модели возможно только как offline pipeline с датасетом, оценкой качества и версионированием модели.

## Что пока нельзя честно назвать закрытым

Полноценный vision/CAD pipeline для произвольных DWG/PDF-планировок еще не закрыт. Сейчас есть безопасный слой: OCR, классификация, rule-based takeoff из текста и видимых чисел, спецификации, evidence graph и запрет на выдуманные объемы. Для настоящей автоматической геометрии нужен отдельный runtime, который умеет читать векторную геометрию, распознавать стены/проемы/помещения, масштаб, слои, экспликации и проверять результат на эталонах.

## Что должна сделать команда

1. Собрать эталонный набор: реальные PDF/PNG/DWG/DXF, спецификации, ведомости, описания проекта и соответствующие сметы из Гранд-Сметы.
2. Для каждого эталона разметить связи: какие страницы/фрагменты дали объем, какая норма применена, какая цена и почему.
3. Подключить CAD/vision runtime: конвертер DWG/DXF в нормализованную геометрию, PDF vector extractor, raster vision для сканов и единый формат takeoff.
4. Сделать benchmark: точность классификации файлов, точность объемов, покрытие нормами, доля ручной проверки, расхождение итоговой стоимости с эталоном.
5. Провести импорт/проверку полной базы норм и цен на production/staging, включая ФСНБ/ФЕР/ТЕР/ФСБЦ и региональные коэффициенты.
6. На staging прогнать сценарий "загрузил планировку + описание -> получил смету -> проверил проблемные позиции -> сохранил смету".
7. После staging QA включать обработку тяжелых наборов только через queue workers с лимитами памяти, таймаутами, retry и мониторингом очередей.

## Критерии готовности следующего этапа

- По типовым планировкам система извлекает площади, периметры, проемы и инженерные элементы не из промпта, а из документа с источниками.
- Каждая платная позиция имеет норму, цену, ресурсный состав или явный статус "требует выбора", без нулевых итогов в готовой смете.
- Дубли не попадают в смету как отдельные работы без основания.
- Пользователь видит не "черный ящик", а цепочку: документ, найденный объем, варианты норм, выбранная цена, причина проверки.
- Качество измеряется на эталонах, а не субъективно по одному примеру.
