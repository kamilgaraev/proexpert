# Project Access Scope

Доступ пользователя к проектам разделен на два независимых слоя:

- роли и кастомные роли определяют, какие действия пользователь может выполнять;
- `project_access_mode` в членстве организации и записи `project_user` определяют, в каких проектах он может это делать.

Режимы доступа:

- `all_projects` - пользователь видит все проекты, доступные его организации;
- `assigned_projects` - пользователь видит только активные проекты, назначенные ему через `project_user`.

Существующие сотрудники при миграции получают `all_projects`, чтобы сохранить текущий доступ. Новые обычные пользователи создаются с `assigned_projects`. Владельцы и администраторы организации создаются с `all_projects`.

Backend enforcement применяется в списке доступных проектов, project-context middleware и сервисе админского доступа к проектным экранам. Для endpoint'ов, которые принимают `project_id` напрямую без project-context маршрута, нужно использовать общий сервис `UserProjectAccessService` или отдельное validation rule на основе этого сервиса.
