# Отдельный обработчик PDF и CAD

## Цель

Вынести распознавание исходных документов МОСТ из общего Horizon-контейнера в отдельный `geometry-worker`, чтобы ошибки, лимиты ресурсов и системная изоляция PDF/DWG не влияли на остальные очереди.

## Архитектура

`ProcessEstimateGenerationDocumentJob` публикуется в Redis-очередь `estimate-generation-documents`. Эту очередь потребляет только контейнер `geometry-worker`, запущенный из того же неизменяемого образа приложения. Остальные задания AI-сметчика продолжают обрабатываться Horizon через очередь `estimate-generation`.

Контейнер получает доступ к Redis, PostgreSQL, S3 и провайдеру распознавания, потому что полный документный сценарий использует эти сервисы. Весь сценарий ограничен ресурсами отдельного read-only контейнера. Процесс LibreDWG дополнительно запускается через `geometry-sandbox`: Landlock разрешает чтение runtime и запись только в каталог конкретного задания, seccomp запрещает создание IPv4/IPv6-сокетов, shell устанавливает лимиты памяти, CPU, размера файлов, числа дескрипторов и времени. PDF runtime проверяется обязательным smoke-тестом и выполняется внутри выделенного контейнера.

## Контейнерные ограничения

- `read_only: true` для корневой файловой системы.
- Отдельный `tmpfs` для `/tmp` с запретом исполнения и setuid.
- `cap_drop: [ALL]` и `no-new-privileges:true`.
- Явные лимиты памяти, CPU и числа процессов.
- Запись Laravel разрешена только в подключённый `storage/logs` и отдельный временный `/tmp`.
- Контейнер не использует `bubblewrap`, привилегированный режим, Docker socket или расширенные capability.

## Развёртывание

До переключения сервисов workflow запускает одноразовый `geometry-worker` с `/usr/local/bin/geometry-runtime-smoke`. Smoke-проверка импортирует PDF runtime, запускает LibreDWG, подтверждает запрет сети и запрет записи вне рабочего каталога. После переключения проверка повторяется внутри работающего `geometry-worker`; при ошибке workflow выводит его журнал и не сообщает об успешной выкладке.

## Совместимость и восстановление

API, модели данных, статусы документов и интерфейс не меняются. Уже завершённые задания не повторяются. Ошибочные документы пользователь повторяет существующей командой; новая попытка публикуется в новую очередь. Необработанные задания старой очереди сохраняются, а новый код обрабатывает последующие повторы идемпотентно по существующим блокировкам документа.

## Проверка

- Контрактный PHPUnit-тест фиксирует имя очереди и единственного потребителя.
- Контракт production runtime запрещает `bubblewrap` и ослабление seccomp/AppArmor.
- Существующие тесты PDF worker, CAD runtime и загрузки документов остаются зелёными.
- Production smoke является обязательным барьером до и после переключения.
