# Правила формирования промптов для фронтенда

Когда пользователь запрашивает "промпт для фронтенда" или ТЗ для реализации функционала на стороне клиента, агент должен составить подробную техническую спецификацию, которая включает:

1. **Список API Endpoints**: Полные пути к ручкам, которые будут использоваться.
2. **Структура запросов (Request Body)**: Подробное описание полей, типов данных и обязательности параметров.
3. **Структура ответов (Response Body)**: Пример успешного ответа и описание возможных ошибок.
4. **Бизнес-логика и рекомендации**:
    - Описание состояний (Loading, Error, Empty).
    - Рекомендации по валидации на стороне клиента.
    - Особенности отображения данных (форматирование дат, валют и т.д.).
    - Взаимодействие между компонентами.

Формат вывода должен быть максимально удобным для того, чтобы пользователь мог скопировать этот текст и передать другому ИИ-агенту или разработчику для реализации фронтенда.
