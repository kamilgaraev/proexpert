# /brainstorm

Этот workflow предназначен для глубокого анализа проблемы и поиска оптимального технического решения через генерацию множества вариантов.

## Шаги брейшторма:

1. **Постановка задачи**
   - Четко сформулируй проблему или цель.
   - Почему это важно сейчас?

2. **Контекст и ограничения**
   - Какие технологии используются?
   - Какие есть ограничения по времени, бюджету или архитектуре?
   - Есть ли уже существующие части кода, которые нужно учитывать?

3. **Генерация идей (Минимум 5)**
   - Предложи как минимум 5 различных подходов к решению задачи.
   - Идеи должны варьироваться от "быстрых и грязных" до "идеальных архитектурных решений".
   - Для каждой идеи кратко опиши суть.

4. **Анализ и сравнение**
   - Создай таблицу сравнения для всех предложенных идей.
   - Критерии для сравнения:
     - Сложность реализации (1-10)
     - Риски (технический долг, безопасность)
     - Время на выполнение
     - Масштабируемость

5. **Выбор и рекомендация**
   - Выбери один (или комбинацию) лучший вариант.
   - Обоснуй свой выбор.
   - Опиши первые конкретные шаги для реализации выбранного варианта.
