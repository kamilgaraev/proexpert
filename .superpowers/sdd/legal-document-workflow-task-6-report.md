# Task 6 — согласование юридических документов

## Результат

Реализован backend-контур согласования документов юридического архива МОСТ:

- версионируемые tenant-scoped шаблоны маршрутов и неизменяемый SHA-256 snapshot на экземпляр;
- последовательные этапы и параллельные группы;
- обязательный юридический этап, защищённый от удаления, отключения, перестановки и смены исполнителя через override;
- привязка решения к точной готовой текущей версии документа и её content hash;
- действия submit, approve, reject, return, reassign, cancel и expire;
- отдельные права RBAC, tenant/project ABAC и разрешение исполнителя по пользователю или роли;
- идемпотентность, optimistic locking и единый порядок блокировок для конкурентных решений;
- неизменяемые решения, аудит в общей транзакции, сроки этапов и механизм reconciliation;
- typed action resolver для UI с человекочитаемыми статусами, разрешёнными действиями и причинами блокировки;
- многофазная PostgreSQL-схема: создание таблиц, concurrent indexes, NOT VALID constraints, последующая валидация и серверные immutable/state-transition guards;
- opt-in PostgreSQL contract test для проверки конкурентной блокировки без подключения к БД по умолчанию.

## Дополнительное усиление после self-review

- Все decision/recovery/action paths проверяют `is_current`, `processing_status=ready`, точный version id и content hash.
- Пользователь с отдельным правом переназначения может разблокировать активный шаг, даже если сам не является текущим исполнителем; approve/reject/return по-прежнему требуют назначения.
- Удалены календарно хрупкие тестовые дедлайны.
